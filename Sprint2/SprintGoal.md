# Sprint broj 2

# Sprint cilj 
 
Razraditi detaljno Product Backlog i definisati User Stories, kako bi tim imao jasno definisane funkcionalnosti potrebne za implementaciju u narednim sprintovima.

# Ključne stavke koje tim želi završiti 

* Razrada postojećih stavki iz Product Backloga
* Kreiranje jasnih i detaljnih User Stories (Naziv story-a, Opis, Poslovna vrijednost, Prioritet, Acceptance Criteria, Pretpostavke / Otvorena pitanja , Veze i zavisnosti)
* Grupisanje User Stories po sprintovima
* Usklađivanje User Stories sa Product Vision i MVP scope-om
* Određivanje pravlnog redoslijeda funkcionalnosti (User Story-a) neophodnog za pravilnu implementaciju
* Definisanje nefunkcionalnih zahtjeva

# Rizici i zavisnosti

*  Nejasno definisan zahtjev može dovesti do loše definisanih User Stories - ako zahtjev nije precizno opisan, članovi tima mogu isti različito razumjeti 

* Pogrešno procijenjena kompleksnost za User Stories - pojedine funkcionalnosti mogu zahtijevati manje ili više vremena od planiranog (manje ili više od 1 sprinta)
